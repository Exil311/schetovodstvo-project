## 1. Необходими програми

Инсталирай следните програми **преди** всичко останало:

| Програма | Версия | Сваляне |
|---|---|---|
| **Node.js** | LTS (v20+) | https://nodejs.org |
| **Python** | 3.x | https://www.python.org/downloads |
| **PostgreSQL** | 15+ | https://www.postgresql.org/download/windows |

> **Забележка:** При инсталация на Python задължително постави отметка на **"Add Python to PATH"** в инсталатора.

---

## 2. Инсталация на PostgreSQL

1. Свали и стартирай инсталатора от https://www.postgresql.org/download/windows
2. По време на инсталацията:
   - **Парола за потребител `postgres`:** `123456` *(или желана парола — трябва да съответства на `.env` файла)*
   - **Порт:** `5432` (по подразбиране)
   - Остави всички останали настройки по подразбиране
3. Завърши инсталацията. **pgAdmin** се инсталира автоматично заедно с PostgreSQL.

---

## 3. Инсталация на Node.js

1. Свали **LTS версията** от https://nodejs.org
2. Стартирай инсталатора и остави всички настройки по подразбиране
3. След инсталацията провери в **Command Prompt (cmd)**:
   ```
   node -v
   npm -v
   ```
   И двете команди трябва да върнат версионен номер.

---

## 4. Инсталация на Python

1. Свали от https://www.python.org/downloads
2. **ВАЖНО:** В началния екран на инсталатора постави отметка на **"Add Python to PATH"**
3. Кликни **"Install Now"**
4. След инсталацията провери в **cmd**:
   ```
   py --version
   ```

## 5. Прехвърляне на проекта

Копирай цялата папка `DormsMaster` на новия компютър в:
```

```

> **Важно:** Пътят трябва да е точно такъв, защото `start.bat` и `LaunchServers.vbs` са написани с абсолютен път. Ако папката е на друго място, ще трябва да се редактират тези два файла.


**`ProgramStarter\LaunchServers.vbs`** — промени на едно място:
```vbs
WshShell.Run chr(34) & "C:\Users\НОВО_ИМЕ\Documents\DormsMaster\ProgramStarter\start.bat" & Chr(34), 0
```

## 6. Настройка на базата данни

### 6.1 Създаване на базата данни

1. Отвори **pgAdmin** (инсталира се с PostgreSQL)
2. Свържи се към локалния сървър с паролата зададена при инсталацията
3. С десен бутон върху **Databases** → **Create** → **Database...**
4. Въведи за **Database name:** `shchetovodstvo`
5. Натисни **Save**

### 6.2 Импортиране на схемата

1. В pgAdmin, кликни с десен бутон на базата `shchetovodstvo` → **Query Tool**
2. Отвори файла `database.sql` от папката на проекта *(File → Open)*
3. Натисни **F5** или бутона ▶ за изпълнение
4. Базата данни е готова

### 6.3 Импортиране на стари данни (при смяна на компютър)

Ако прехвърляш данни от стар компютър:

**На стария компютър** — направи backup:
1. В pgAdmin → с десен бутон на `shchetovodstvo` → **Backup...**
2. Формат: **Custom** или **Plain**
3. Запази файла (напр. `backup.sql`)

**На новия компютър** — възстанови:
1. Създай базата `shchetovodstvo` (виж стъпка 6.1)
2. С десен бутон на `shchetovodstvo` → **Restore...**
3. Избери backup файла и натисни **Restore**

## 7. Настройка на .env файла

Файлът `.env` се намира в основната папка на проекта. Провери дали настройките съответстват на твоята инсталация:

```env
DATABASE_URL=postgresql://postgres:123456@localhost:5432/shchetovodstvo
SESSION_KEY=misho123
PORT=3000
```

| Поле | Описание |
|---|---|
| `postgres` | Потребителско име на PostgreSQL (по подразбиране е `postgres`) |
| `123456` | Паролата, зададена при инсталацията на PostgreSQL |
| `localhost` | Хост (не се променя) |
| `5432` | Порт (не се променя) |
| `shchetovodstvo` | Името на базата данни |
| `SESSION_KEY` | Таен ключ за сесиите (може да остане същия) |
| `PORT` | Портът, на който работи приложението |

> Ако паролата ти е различна, смени `123456` с твоята парола.

## 10. Стартиране с ProgramStarter (препоръчително)

Папката `ProgramStarter` съдържа скриптове, които стартират **и двата сървъра автоматично** и отварят браузъра.

### Файлове в ProgramStarter:

| Файл | Описание |
|---|---|
| `start.bat` | Основен скрипт — стартира Node.js и Python сървърите, отваря браузъра |
| `LaunchServers.vbs` | Стартира `start.bat` **без черен прозорец** (скрит режим) |
| ` .lnk` | Shortcut файл за Desktop |

### Как работи:

1. `LaunchServers.vbs` стартира `start.bat` невидимо (без черен cmd прозорец)
2. `start.bat` пуска Node.js и Python сървърите като фонови процеси
3. След 5 секунди автоматично отваря `http://localhost:3000` в браузъра
4. Ако някой от сървърите спре/падне — **автоматично се рестартира**

## 11. Създаване на Desktop shortcut

### Вариант А: Копиране на готовия shortcut

В папката `ProgramStarter` има файл ` .lnk` (shortcut). Можеш да го **копираш директно на Desktop**.

1. С десен бутон върху Desktop → **New** → **Shortcut**
2. В полето **"Type the location of the item"** въведи:
   ```

   ```
   *(Смени `Kasa` с твоето потребителско име ако е различно)*
3. Натисни **Next**
4. Въведи името: `DormsMaster`
5. Натисни **Finish**

## 13. Google Apps Script интеграция

Програмата използва **Google Apps Script** като облачна услуга за:
- Експортиране на справки в **Google Docs** и **Google Sheets (Excel)**
- Генериране на **заповеди за настаняване** и **обходни листове**
- Автоматичен месечен **backup на базата данни в Google Drive**

### Как работи

```
ДормsMaster (Node.js) ──POST──► Google Apps Script Web App ──► Google Drive / Docs / Sheets
```

Скриптът е публикуван като **Web App** в Google акаунта, обвързан с програмата. Текущият URL на скрипта е вграден в кода на приложението.

### Текущ Google акаунт и Script URL

Текущият Apps Script URL (вграден в кода):
```
https://script.google.com/macros/s/AKfycbxBgFuipuCatEXmRvRCK_Q4KMGecrXN4wXfhiBf5l_POmG7-kX6O9k2qWSjN57DGnPgYA/exec
```

> **Важно:** Документите, справките и архивите се записват в Google Drive на акаунта, от който е публикуван скриптът.

### При смяна на Google акаунт — стъпки за пренастройка

#### Стъпка 1: Качи скрипта в новия акаунт

1. Влез в новия Google акаунт
2. Отиди на https://script.google.com
3. Кликни **New Project**
4. Изтрий съдържанието на редактора и постави целия код от файла `google_docs_export_script.gs`
5. Запази проекта (Ctrl+S) — дай му произволно име, напр. `DormsMaster`

#### Стъпка 2: Авторизирай скрипта

1. В редактора избери функцията `authorizeMe` от падащото меню
2. Кликни **Run** (▶)
3. Ще се появи прозорец за разрешения — кликни **Review permissions** → избери акаунта → **Allow**
4. Трябва да видиш в логовете: `Permissions granted!`

#### Стъпка 3: Публикувай като Web App

1. Кликни **Deploy** → **New deployment**
2. Кликни иконата ⚙️ до "Select type" → избери **Web app**
3. Настрой:
   - **Description:** DormsMaster
   - **Execute as:** `Me` (твоят акаунт)
   - **Who has access:** `Anyone` *(задължително — иначе Node.js сървърът не може да извика скрипта)*
4. Кликни **Deploy**
5. Копирай новия **Web app URL** — ще изглежда така:
   ```
   https://script.google.com/macros/s/НОВ_ID_ТУК/exec
   ```

#### Стъпка 4: Смени URL-а в кода на приложението

Отвори следните файлове и замени стария URL с новия на **всички места**:

| Файл | Брой замени |
|---|---|
| `public/js/reports.js` | 3 места |
| `public/js/settings.js` | 8 места |
| `public/js/students.js` | 2 места |

Можеш да използваш **Find & Replace** (Ctrl+H) в текстов редактор:
- **Намери:** `AKfycbxBgFuipuCatEXmRvRCK_Q4KMGecrXN4wXfhiBf5l_POmG7-kX6O9k2qWSjN57DGnPgYA`
- **Замени с:** *(новото ID от новия URL)*

> Само ID частта между `/s/` и `/exec` се различава между deployment-ите.

## 14. Справочник с грешки

### ❌ `Error: connect ECONNREFUSED 127.0.0.1:5432`
**Причина:** PostgreSQL не е стартиран.  
**Решение:** Отвори **Services** (Windows + R → `services.msc`) и стартирай `postgresql-x64-XX` сервиза.

### ❌ `password authentication failed for user "postgres"`
**Причина:** Грешна парола в `.env` файла.  
**Решение:** Провери паролата в `.env` — трябва да съответства на паролата зададена при инсталацията на PostgreSQL.

### ❌ `database "shchetovodstvo" does not exist`
**Причина:** Базата данни не е създадена.  
**Решение:** Следвай [Стъпка 6](#6-настройка-на-базата-данни).

### ❌ `'node' is not recognized as an internal or external command`
**Причина:** Node.js не е инсталиран или не е в PATH.  
**Решение:** Преинсталирай Node.js от https://nodejs.org и рестартирай компютъра.

### ❌ `'py' is not recognized as an internal or external command`
**Причина:** Python не е инсталиран или не е добавен в PATH.  
**Решение:** Преинсталирай Python и постави отметка на **"Add Python to PATH"**.

### ❌ `Cannot find module` (при `npm start`)
**Причина:** `node_modules` папката липсва.  
**Решение:** Изпълни `npm install` в папката на проекта.

### ❌ Браузърът се отваря но страницата не зарежда
**Причина:** Сървърите не са се стартирали навреме.  
**Решение:** Изчакай 10-15 секунди и презареди страницата (F5).
